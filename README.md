## File Structure:

<!-- Here is what each part likely means. -->

### `_config.yml`
This is the main Jekyll configuration file. It contains:

* site title
* description
* theme
* base URL / site URL
* plugins
* author info

<!-- This file controls the global behavior of the site. -->

### `_posts/*` Files(Important!!!)
This directory contains the **blog pages**. In Jekyll, posts live inside `_posts` and follow the naming pattern:

`YYYY-MM-DD-title.markdown`

<!-- So this means you have at least one post called something like “welcome,” dated March 26, 2025. -->

### `404.html`
This is the custom “Page not found” page. If someone visits a broken link on this site, GitHub Pages shows this page.

### `about.markdown`
This is standalone page, accessible at:
        [https://om-nain.github.io/about.html](https://om-nain.github.io/about.html)

<!-- `/about/`

It often contains a short intro about you or the site. -->

### `Gemfile` and `Gemfile.lock`
These are Ruby/Bundler files. They define the Jekyll dependencies used to build the site locally.
GitHub Pages sites built with Jekyll often have these so you can run the blog on your machine.

### `index.markdown`
This is the homepage.
Depending on the theme and front matter inside it, this page may:

* show recent posts
* show a blog landing page
* show some custom intro text
